# 🚀 Contributing to SpaceDMV

Thank you for your interest in contributing to **SpaceDMV**, an interplanetary, space‑government themed DMV system built with Next.js, TypeScript, and TailwindCSS.  
This project is part of the larger **LEAF Universe**, and contributions help expand a futuristic ecosystem of civilian‑safe, transparent, and accessible technology.

This document outlines how to contribute effectively and responsibly.

## 1. 📜 Code of Conduct
All contributors must follow the project’s **CODE_OF_CONDUCT.md**.  
Respect, professionalism, and constructive collaboration are required at all times.

---

## 2. 🛠️ How to Contribute

You can contribute in several ways:

### 🧩 Features & Enhancements
Propose new UI components, workflows, or improvements to existing modules.

### 🐛 Bug Reports
Report issues with clear reproduction steps.

### 📚 Documentation
Improve or expand the `/docs` folder.

### 🎨 UI/UX Enhancements
Refine styling, animations, accessibility, or layout consistency.

### 🔒 Security Improvements
Report vulnerabilities responsibly (see Section 10).

---

## 3. 🧪 Development Workflow

1. Fork the repository  
2. Create a feature branch  
3. Make changes following coding standards  
4. Test your changes locally  
5. Submit a pull request  

---

## 4. 🌿 Branching Strategy

Use the following naming conventions:

- `main` — stable production branch  
- `dev` — active development  
- `feature/<name>` — new features  
- `fix/<name>` — bug fixes  
- `docs/<name>` — documentation updates  

Examples:  
- `feature/svin-wizard-validation`  
- `fix/appointment-calendar-bug`
